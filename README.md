## 🎯 解决的问题

> 大型语言模型（LLM）在执行精确的数据分析任务时，普遍存在可靠性不足、容易产生幻觉、且缺乏专业判断力的问题。这阻碍了它们成为真正可用的商业智能工具。

**InsightAgent** 通过设计一套“分层防御”的 AI Agent 架构，并建立了一套数据驱动的迭代与评测体系，成功地将一个通用的 LLM，改造为了一个高鲁棒性的专业数据分析 Agent。

## ✨ 核心特性

*   **🧠 智能规划**：自动将复杂的用户请求分解为简单的、可执行的 Python/SQL 步骤。
*   **🤖 自我修正**：执行过程中遇到错误（如语法、逻辑错误）时，能自动进行多达3次的尝试与修正。
*   **📊 专业可视化**：能根据分析结果，智能选择最佳的图表类型（条形图、折线图等）进行展示。
*   **🛡️ 鲁棒可靠**：内置“资深分析师”的专业知识，能识别并拒绝不合理的数据请求（如分析不存在的列）。
*   **📖 格式自适应**：能根据查询结果的类型，智能选择最佳的输出格式（Markdown表格、自然语言总结等）。

## 🏛️ 架构设计哲学：构建“可信AI”的三层防御

我们的核心理念是：**用确定性的代码框架，去驾驭不确定性的 AI 模型。**

---

### 🛡️ 第一层防御【事前预防】：“计划-执行”架构

*   **解决的问题**：“组合复杂性”导致的 LLM 认知过载与高失败率。
*   **我们的方案**：将用户的复杂任务解耦为线性的、简单的代码步骤列表。这从源头上极大地降低了单步执行的失败概率，使整个流程变得清晰、可控。

---

### 🔧 第二层防御【事中控制】：AI 自我修正循环

*   **解决的问题**：LLM 内在的、无法避免的“不确定性”和随机性错误。
*   **我们的方案**：为每一个执行步骤都配备了一个带3次重试机会的“AI 自我修正”安全网。当代码执行失败时，Agent 会分析错误信息，生成修正后的代码，并重新尝试。

我们通过**“故障注入测试”**，主动模拟了常见的语法和逻辑错误，**验证了该“安全网”的有效性**。

---

### 🎓 第三层防御【认知注入】：“资深分析师”的专业知识

*   **解决的问题**：AI 缺乏人类的“常识”和“专业判断力”，有时会导致“愚蠢”的错误。
*   **我们的方案**：通过在 Prompt 中系统性地注入专业准则（如“数据可视化最佳实践”、“列存在性检查”等），让 Agent 学会了拒绝不合理请求，并能像真正的分析师一样选择最佳的数据呈现方式。

## 🛠️ 如何运行

1.  **克隆仓库**
    ```bash
    git clone https://github.com/your-username/InsightAgent.git
    cd InsightAgent
    ```

2.  **安装依赖**
    ```bash
    pip install -r requirements.txt
    ```

3.  **设置环境变量**
    *   复制 `.env.example` 文件为 `.env`
    *   在 `.env` 文件中填入您的 ZHIPUAI_API_KEY
    ```
    ZHIPUAI_API_KEY="your_api_key_here"
    ```

4.  **启动应用**
    ```bash
    streamlit run app.py
    ```

> **洞察三：AI时代的产品角色。** 我们不再仅仅是“需求的翻译官”，更是“AI行为的设计师”和“机器智能的驯兽师”。
